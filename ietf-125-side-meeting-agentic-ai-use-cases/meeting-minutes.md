## IETF 125 Side Meeting – Agentic AI Use Cases (Summary Minutes) 

The meeting identified a strong need to bring structure to the growing set of agentic AI use cases discussed at the IETF. Participants agreed on the following goals:

- Define a taxonomy for agentic AI use cases.

- Develop templates and a guidance checklist so use cases are consistently described and usable for gap analysis and requirements work.

- Use the outcomes to inform future BOFs and WGs related to agentic AI.

It was emphasized that this effort must not slow down specification work:

- There is urgency to progress agentic AI specifications.

- Where scope and consensus already exist, specifications can proceed in parallel.

- The use‑case effort is intended to complement ongoing work and help reassess existing and future specifications.

Several use cases were presented, prompting discussion on potential new features and early gap analysis.

In the final discussion, participants raised the following points:

- A lightweight curation or gatekeeping role may be needed for accepting use cases into a central document.

- The taxonomy should organize use cases by functional domains (e.g., discovery, identity, security, transport).

- Until a WG is formed, the work should proceed organically with interested contributors.

- Requirements from other SDOs (e.g., 3GPP) can be considered.

- Use‑case templates should include quantitative properties (e.g., number of agents, users).

- Aim to have an initial document with the taxonomy, properties and template in June, to collect feedback 1 month before IETF 126. 

A proposed approach is to maintain:

- A base document defining the taxonomy, templates, and properties.

- Individual drafts per use case, written using the common structure.

- A summary table in the base document referencing all use‑case drafts.
